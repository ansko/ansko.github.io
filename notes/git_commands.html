Команды git

Некоторые, которые я чаще всего использую.
  • add
    • добавить в индекс содержимое файлов
    -A --all - обновляется весь индекс (или по пути, если передан путь)
  • checkout
    • переключение между ветками и восстановление файлов
    -b - если ветки нет, то она создаётся
  • clone
    • склонировать репозиторий в новую папку -b --branch - в качестве указателя HEAD указанная ветка
  • commit
    • создать новый коммит с текущим содержимым индекса; новый коммит являет непосредственным потомком HEAD, текущая ветка обновляется так, чтобы указывать на этот коммит
    -a --all - автоматически добавить обновлённые и удалённые файлы в коммит, но не новые
    -F --file - взять сообщение для коммита из файла (-) - из стандартного потока ввода
    -m --message - использовать переданное сообщение для коммита
  • config
    • посмотреть и изменить свойства репозитория или общие свойства самого git
    -l --list - вывести все установленные в конфигурационном файле переменные вместе с их значениями
    --global - действия для всех репозиториев (установка переменных, --list, ...)
    --local - действия для текущего репозитория (опция по умолчанию)
    переменная user.email - почта коммитящего пользователя
    переменная user.name - имя коммитящего пользователя
    переменная remote.origin.url - url удалённого репозитория
  • init
    • Создать пустой репозиторий или переинициализировать репозиторий
  • merge
    • слияние историй изменений
  • pull
    • получить данные (из удалённого репозитория или локальной ветеки) и выполнить слияние с ним
  • push
    • обновить удалённый рпепозиторий из локального
    --all - обновить все ветки
  • status
    • показать статус локального репозитория
